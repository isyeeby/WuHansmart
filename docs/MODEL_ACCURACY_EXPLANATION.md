# 模型精度与数据标签说明

## 📊 R² = 0.9619 是怎么计算的？

### 1. R² (决定系数) 计算公式

```
R² = 1 - (SS_res / SS_tot)

其中：
- SS_res = Σ(y_actual - y_predicted)²  (残差平方和)
- SS_tot = Σ(y_actual - y_mean)²       (总平方和)
```

### 2. 具体含义

**R² = 0.9619 表示：**
- ✅ 模型能解释 **96.19%** 的房价变异
- ✅ 预测值与真实值高度相关
- ✅ 残差（误差）很小

**举例说明：**
```
真实房价：300 元
模型预测：295 元
误差：5 元 (1.67%)
```

### 3. 测试集评估过程

训练时数据划分：
- **训练集**: 80% (4000 条)
- **测试集**: 20% (1000 条)

评估指标：
| 指标 | 数值 | 含义 |
|------|------|------|
| **R²** | 0.9619 | 拟合度 96.19% |
| **MAE** | 24.98 元 | 平均绝对误差 |
| **RMSE** | 31.33 元 | 均方根误差 |

这意味着：
- 预测 100 个房源，平均误差约 **25 元**
- 对于 300 元的房价，误差率约 **8.3%**

### ⚠️ 标签缺失的影响

**1. 训练数据量不足**
```
理想情况：5000 条带标签数据
当前情况：2000 条带标签数据
影响：模型可能欠拟合，泛化能力下降
```

**2. 数据偏差风险**
```
如果 2000 条带标签数据不是随机分布：
- 可能集中在某些商圈
- 可能集中在某些价格区间
- 导致模型偏向这些区域
```

**3. 模型精度虚高可能**
```
当前 R² = 0.9619 可能偏高的原因：
1. 使用模拟数据训练（5000 条完整标签）
2. 真实数据只有 2000 条，评估不充分
3. 测试集可能不是独立分布
```
