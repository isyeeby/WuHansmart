一、毕业设计进展情况

本项目旨在基于途家武汉区域样本，完成民宿价格采集、Hive 离线分析、机器学习与推荐及 Web 一体化展示，支撑驾驶舱、智能定价、投资机会与个性化房源等业务。与任务书中的需求分析、架构设计、采集入库、特征与数仓、可视化与模型服务等要求总体对应。自开题以来，已按计划完成以下工作：

技术学习与框架搭建：系统采用前后端分离架构。后端基于 FastAPI 提供 RESTful 接口，联机数据使用 MySQL，分析类接口支持 Hive 优先、失败时回退 MySQL 同口径聚合；数据处理与脚本侧使用 Python、Pandas 等完成清洗与导入。前端基于 React、TypeScript、Ant Design 与 ECharts 实现页面与图表，开发阶段通过 Vite 代理联调，生产环境前端构建产物与后端 API 可由 Nginx 反向代理统一对外提供服务。用户侧采用 JWT 鉴权，并支持问卷与个人信息等联机数据写入。

核心功能开发：
1. 数据链路：基于浏览器自动化与接口监听完成列表、详情与价格日历等采集与持久化，支持分文件存储与断点续传思路，便于增量更新与异常恢复；导入脚本将房源主数据、日历与标签等写入业务库，并可按设计导入 Hive 分层表，支撑行政区、商圈、设施等多维统计与驾驶舱指标。
2. 分析与服务模块：实现经营驾驶舱（KPI、趋势、热力等）、数据分析、价格预测、个性化推荐与首页推荐、投资沙盘与价格机会、房源列表与详情（含关键词检索与登录态个性化排序）、多房源对比、我的房源与竞品参照、收藏与浏览历史、个人中心等后端接口与前端页面；提供 ApiTest 等页面便于联调与烟测。
3. 模型与策略：接入 XGBoost 定价推理与特征工程管线；推荐侧以显式条件匹配为主，结合内容相似度矩阵与浏览、收藏等行为加权；文本侧可按设计输出场景等辅助分数供排序使用；在模型或矩阵文件缺失时按约定降级为热度、评分等兜底策略，保证服务可用。
系统测试与优化：已编写后端路由烟测（pytest），修复部分接口与类型边界问题；维护模块说明、数据层与 API 口径类文档，便于中期验收、线上排错与毕业论文撰写。

二、已获得阶段性成果

在工程形态上已形成「采集—清洗导入—Hive 离线汇总—联机查询与模型推理—前端可视化」的完整链路，能够支撑课题设定的经营分析、定价辅助与房源发现等场景。数据从采集、入库到 Hive 聚合基本打通；驾驶舱与分析类指标可在不同部署条件下稳定输出或在 Hive 不可用时回退至 MySQL 同口径结果。核心业务页面与 API 已联调可用；定价与推荐能力已接入服务并在缺失资源时自动降级。系统已部署至线上服务器，经 Nginx 等配置对外提供访问，便于指导教师检查与中期演示。已具备 pytest 与前端烟测、模块说明等可复现材料，为后续论文中的系统实现章节与截图、实验描述打下基础。

三、存在的问题

1. 数据工程：增量更新节奏、手工与定时导数流程仍需制度化；缺失与异常字段的巡检规则需固化，部分衍生字段与 Hive 汇总需补充校验脚本，避免多次导数后前后端指标不一致。
2. 模型与实验：定价模型需完成规范交叉验证与独立测试集评估，将 MAE、RMSE、MAPE 等指标与超参数、特征列表一并整理进论文；推荐与列表重排策略需补充可复现的对比或消融说明，便于答辩时回应「效果依据」。
3. 系统性能：房源列表、驾驶舱与复杂聚合接口在样本量继续增大时，仍有索引与 SQL 优化空间；可视课程要求对热点只读结果引入短周期缓存，并记录轻量压测或慢接口监控结果。
4. 功能与体验：竞品参照当前以同区与距离为主，待增加卧室数、可住人数等筛选以更贴近业务语义；投资、机会与预测等模块的参数说明与结果释义需与后端字段 glossary 对齐；部分页面空态、加载与操作引导仍需统一风格。
5. 运维与材料：线上环境依赖配置、健康检查与日志查看方式需写入部署说明；接口清单、训练日志与实验记录需集中归档，便于论文附录与复现说明。

四、下一步的计划安排

1. 数据与特征：落实增量更新与导数清单（含失败重试与校验项），将异常与缺失规则写入文档；补齐与 Hive 汇总字段一致的对账脚本，必要时抽样比对驾驶舱与列表口径。
2. 模型与实验：完成定价模型 K 折或留出集评估，按行政区或商圈分层汇报误差；整理推荐与列表策略的实验表格或截图；视篇幅增加残差分布、特征重要性等图表支撑论述。
3. 性能优化：对慢 SQL 做 explain 分析与索引调整，列表与分页参数默认值合理化；对驾驶舱等只读聚合评估缓存 TTL 与失效策略，并保留优化前后对比记录。
4. 功能收尾：实现竞品多条件筛选与文案统一；走查投资、机会、预测、我的房源等核心路径的边界与空态；保持前后端版本标签与变更记录可查。
5. 部署与运维：完善线上部署说明（环境变量、Nginx、进程守护与健康检查），规范日志路径与常见故障排查步骤。
6. 论文与答辩：在第八学期进度内完成论文初稿，撰写系统实现、实验与测试章节，插入关键界面与架构图；整理答辩演示顺序与烟测清单，确保线上环境可稳定演示。
