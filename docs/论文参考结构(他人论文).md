国内许多学者从不同角度、不同类型的研究中发现SSM 框架。如《SSM 框架的图书管理系统的设计与实现》[1]，通过本文对SSM框架的介绍，提出一种基于SSM框架的图书管理系统的设计方案，运用SSM进行对象管理，使用Spring进行前端和后端的数据传输工作；利用MyBatis作为数据操作层，完成各种数据库查询的操作。该方案可以为其他SSM框架下的系统开发提供很好的借鉴作用。

1.4  技术关键点
Spring MVC（Model-View-Controller）是基于Java的Web应用程序开发框架，它是Spring Framework的一部分，用于构建灵活、可扩展的现代Web应用程序。它通过将应用程序分解为模型、视图和控制器三个组件，提供了一种结构良好的方法来处理Web请求和响应。Spring MVC在现代Web应用程序中扮演着重要的角色。它不仅帮助开发人员将应用程序逻辑和用户界面分离，还提供了一些关键功能，包括： 分层架构、请求处理和路由、视图解析、数据绑定与验证、拦截器与过滤器。
MyBatis 是一款优秀的持久层框架，它支持定制化 SQL、存储过程以及高级映射。MyBatis 避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集。MyBatis 可以使用简单的 XML 或注解来配置和映射原生信息，将接口和 Java 的 POJOs(Plain Ordinary Java Object,普通的 Java对象)映成数据库中的记录。
简单的来说就是：MyBatis是一个实现数据持久化的开源框架，简单理解为就是对JDBC进行封装。开发者只需要关注sql语句本身，而不是去花费精力去处理加载驱动、创建连接、创建statement等繁杂的过程。
MySQL数据库是数据库工具里最受认可的其中一个应用软件。需要说明的信息就是，本程序的开发就运用到了此数据库。它将程序数据通过使用不同的数据表格进行保存，在增加了程序数据的存储速度的时候，也提高了数据库的灵活性。

2.1  系统可行性分析
2.1.1  技术可行性分析

6  总结
本次系统基于SSM框架，结合当前电商行业蓬勃发展的趋势以及消费者对鸿星尔克产品线上购物体验的需求升级，深入分析同类商城的研究现状和背景，从用户实际使用场景出发，精心设计并实现了相关功能模块，打造出了一个集商品展示、交易、评价、发/收货、退款于一体的鸿星尔克运动商品销售系统。
系统的创新在于使用了多源数据协同处理的方法（分析获取到的所有信息全部综合到一起，并对信息进行统一的评价，最后得到统一的信息的技术），拓宽了可参考信息的宽度。在之前的研究中，这种方法没有被充分利用。这种方法的有效性为之前的研究做了很大的补充与完善。
鸿星尔克运动商品销售系统现已完成了开发，但是受限于个人技术能力水平和开发时间，除了基本功能可以符合用户需求外，在页面设计层面上没有融入更多时尚、美观、多元化的元素，需要从美学角度进行优化。在项目的代码中，存在许多不必要的复杂和重复部分，需要对其进行优化和重构，以使代码更加简洁和高效。在项目的功能实现方面，项目的功能不够丰富。需要扩展更多的功能。
未来研究方向可以进一步优化系统性能，采用分布式架构，提升系统的可扩展性。集成大数据分析与AI技术，为用户提供个性化推荐和精准营销功能。进一步完善系统的移动端支持，增强用户在移动设备上的购物体验。
