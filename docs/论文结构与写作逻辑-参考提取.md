## 四、整体论证逻辑（为什么这么排）

**一条主链：**

> **问题与价值 → 能不能做、要什么 → 总体方案 → 逐项落地 → 验证 → 收束**

对应关系：

- **绪论**：为什么做（背景、意义）→ 别人做到哪（国内外与趋势）→ 本文做什么、用什么技术路线（目标与思路）。
- **系统分析**：可行性堵住「空想」质疑；需求与用例把「要做什么」说清楚、可对照实现。
- **总体设计**：架构定分层；模块图与需求对齐；数据库从概念到表，为详细实现提供依据。
- **详细设计与实现**：按角色、按业务流程展开，对应第 2 章需求与用例。
- **系统测试**：用例表映射功能，结论为「满足需求、可正常使用」。
- **总结**：回扣绪论中的目标，写创新、不足与未来工作。

**可复用的一句话：**

先让读者相信**值得做且能做**，再说明**要做什么**，然后给出**整体怎么搭**，再按**谁用、做什么事**写实现，用测试**证明对上需求**，最后用总结**回扣开头**。
