## 一、产品概述

### 1.1 产品定位
基于机器学习算法的民宿投资智能决策平台，为民宿投资者和经营者提供数据驱动的选址、定价、运营优化建议。

### 1.2 核心AI能力
| 模型 | 技术实现 | 应用场景 |
|------|----------|----------|
| **日级 XGBoost 定价模型** | 日历样本 + 静态特征；Booster 回归（参数见训练脚本与环境变量） | 智能定价锚定日、14 天曲线、因子分解、价格机会（配合行政区兜底） |
| **智能推荐（内容相似度为主）** | 多维房源特征 + SVD 降维 + 余弦相似度；可选融合隐式行为相似度 | 相似房源、条件推荐、登录用户历史扩展、竞争分析 |

### 1.3 数据基础
- **房源数据**: 4,300条，包含title、district、tags、price、rating、comment_count等字段
- **价格日历**: 187天历史价格数据
- **特征工程**: 从title解析户型（卧室数、床位数、面积），从tags提取设施特征

### 1.4 房源列表与详情（联机 MySQL）
- **筛选**：紧凑型筛选区（行政区、卧室、价格区间、排序）；支持 **关键词** 检索标题/行政区/商圈（`GET /api/listings`，Query：`keyword`）。
- **个性化排序**：请求头携带有效 JWT 时可选用 `sort_by=personalized`：根据用户 **收藏（权重 3）** 与 **浏览历史（权重 1）** 统计偏好行政区、商圈（各 Top5），列表按匹配度优先、再按平台收藏数降序；未登录、令牌无效或无行为数据时与按收藏数排序一致。
- **原理说明**：**规则重排**（SQL `CASE` 区域匹配分 + 收藏数次序），**不经过** `GET /api/recommend` 与相似度矩阵；与推荐页、首页条为**独立路径**。详见 [`LISTINGS_PERSONALIZED_SORT.md`](./LISTINGS_PERSONALIZED_SORT.md)。
- **浏览历史**：用户打开房源详情并成功加载数据后，前端对登录用户调用 `POST /api/user/me/history` 写入 `user_view_history`，为上述排序与推荐模块提供行为信号。

## 三、详细功能需求

### 【模块一】投资选址（5个功能）

#### 功能1: 区域投资热力图
**功能描述**: 可视化展示各区域的民宿投资热度分布

**输入数据**:
- district（区域）
- price（价格）
- comment_count（评论数，作为销量代理指标）

**核心指标**:
- 区域平均房价
- 区域平均入住率（通过评论增量估算）
- 区域房源密度

**输出**:
- 热力地图（颜色深浅表示投资热度）
- 热门区域排行榜
- 区域筛选器（价格区间、户型）

---

#### 功能2: 商圈竞争力分析
**功能描述**: 分析指定商圈的竞争格局和市场饱和度

**输入数据**:
- district
- price
- rating
- tags（设施配置）

**分析方法**:
- 统计商圈内房源数量、平均价格、平均评分
- 计算价格离散度（竞争程度指标）
- 设施配置分布分析

**输出**:
- 商圈竞争力评分（1-10分）
- 同商圈房源价格分布图
- 市场饱和度预警

---

#### 功能3: 价格洼地挖掘
**功能描述**: 识别价格低于市场价值的潜力区域

**技术实现**:
- 使用**XGBoost模型**预测各房源的理论价格
- 对比实际价格与预测价格
- 识别价格差异>20%的"洼地"房源

**输出**:
- 价格洼地地图
- 洼地房源列表（含理论价格vs实际价格对比）
- 洼地成因分析（设施不足、位置偏僻等）

---

#### 功能4: 投资收益率排行榜
**功能描述**: 基于ROI模型的区域投资收益率排名

**技术实现**:
- 整合**XGBoost价格预测模型**预测收益
- 结合运营成本估算（基于面积、卧室数）
- ROI = 年预测收益 / (装修成本 + 运营成本)

**核心指标**:
- 年收益预测（基于XGBoost价格预测）
- 投资回报率（ROI）
- 回本周期估算

**输出**:
- ROI排行榜（Top 50区域）
- 投资推荐标签（"高收益"、"稳健型"、"高风险高回报"）

---

#### 功能5: 房源详情深度分析
**功能描述**: 单个房源的全维度分析报告

**输入数据**: 房源所有字段

**分析内容**:
1. 基础画像：户型、面积、设施清单
2. 价格分析：历史价格走势、与同类房源对比
3. **相似度推荐**：相似房源与个性化扩展（内容矩阵为主，可选行为增强）
4. 竞品分析：同区域同户型竞争房源

**输出**:
- 房源评分卡片
- 价格竞争力指数
- 相似房源推荐列表（基于预计算相似度矩阵）

#### 功能15: 投资计算器 ⭐XGBoost核心功能
**功能描述**: 计算民宿投资的ROI和回本周期

**技术实现**:
- **XGBoost模型**预测月收益
- 输入：区域、户型、设施、运营成本
- 计算：年收益、ROI、回本周期

**计算公式**:
```
月收益预测 = XGBoost(区域, 卧室数, 面积, 设施, 评分预测)
年运营成本 = 月租金 + 水电 + 保洁 + 平台佣金
ROI = (年收益预测 - 年运营成本) / 装修投资成本
回本周期 = 装修投资成本 / 年净收益
```

**输出**:
- 投资收益预测报告
- 敏感性分析（"如果入住率下降10%，ROI变化多少？"）
- 投资建议（"推荐投资"/"谨慎投资"/"不建议投资"）

## 四、机器学习模型规格

### 4.1 日级 XGBoost 定价模型

**实现要点**（与代码一致）：

- **训练**：`scripts/train_model_daily_mysql.py`；样本为「房源 × 日历日」；目标 `log1p(价格)`；时间切分 train/val/test。
- **推理**：`app/services/daily_price_service.py` 加载 `xgboost_price_daily_model.pkl` 与 `feature_names_daily.json` 等；`PredictionRequest` → 逐日曲线与锚定日 `base_price`。
- **特征**：静态房源字段 + 区位编码与聚合统计 + 日期/节假日/horizon（见 `app/ml/daily_calendar_features.py`）。

**评估指标**（训练日志与 `model_metrics_daily.json`）：

- MAE、RMSE、R²、MAPE（原始价格尺度）

### 4.2 智能推荐系统（内容相似度为主）

**模型架构**:
```python
class ContentSimilarityRecommender:  # 示例类名；实现见 app/services/recommender.py + build_recommendation_model.py
    def __init__(self):
        self.item_features = None
        self.item_features_scaled = None
        self.similarity_matrix = None
        self.listing_ids = None

    def prepare_item_features(self, features_df):
        """构建房源特征向量"""
        features = [
            'district_encoded',    # 区域
            'bedroom_count',       # 卧室数
            'area_sqm',            # 面积
            'facility_count',      # 设施数量
            'rating',              # 评分
            'price_normalized'     # 归一化价格
        ]
        self.item_features = features_df[features].values
        self.item_features_scaled = StandardScaler().fit_transform(self.item_features)

    def build_similarity_matrix(self):
        """构建余弦相似度矩阵"""
        self.similarity_matrix = cosine_similarity(self.item_features_scaled)

    def get_similar_listings(self, unit_id, n_recommendations=5):
        """获取相似房源"""
        idx = self.listing_id_to_idx[unit_id]
        sim_scores = list(enumerate(self.similarity_matrix[idx]))
        sim_scores = sorted(sim_scores, key=lambda x: x[1], reverse=True)
        similar_indices = [i[0] for i in sim_scores[1:n_recommendations+1]]
        return [self.listing_ids[i] for i in similar_indices]

    def recommend_for_user(self, viewed_listings, n_recommendations=10):
        """基于浏览历史推荐"""
        scores = np.zeros(len(self.listing_ids))
        for listing_id in viewed_listings:
            idx = self.listing_id_to_idx[listing_id]
            scores += self.similarity_matrix[idx]

        # 排除已浏览的房源
        for listing_id in viewed_listings:
            scores[self.listing_id_to_idx[listing_id]] = -1

        top_indices = np.argsort(scores)[::-1][:n_recommendations]
        return [self.listing_ids[i] for i in top_indices]
```

**评估指标**:
- 推荐准确率（Precision@K）
- 推荐多样性
- 冷启动覆盖率
